設定 (Settings)

アドオンの各種パラメータおよびキャラクター定義を変更できます

キャラクター定義 (character.md)

エージェントの性格、使命、話し方、行動指針を規定する Markdown テキストです。保存すると次回のエージェント起動時から適用されます。

会話・独り言のフキダシ上部に表示される名前です。

追加コンテキスト (extra_context.conf)

毎回のLLM実行時、システムプロンプトの直後に追加される動的なテキスト（コンテキスト）を出力するシェルコマンドを1行に1個記述します。

在宅判定 (presence)

居住者が在宅しているか判定するためのエンティティを設定します。

行動ポリシー (policies)

エージェントの行動指針やルールを箇条書きで設定します。

▶

カメラ (cameras)

カメラとマイクは部屋の感覚として扱い、部屋への侵入や減衰の影響を受けます。HAの camera エンティティ、または go2rtc のストリーム名を指定します。

部屋 (room) ソース (source) ラベル (label) 編集 削除

マイク (mics)

カメラとマイクは部屋の感覚として扱い、部屋への侵入や減衰の影響を受けます。実マイクや環境音の入力元を指定し、STT を使うかどうかを切り替えます。

部屋 (room) ソース (source) ラベル (label) 編集 削除
▶

映像ソース (video_media)

映像ソースと音声ソースはメディア受信として扱い、侵入不要でコンテンツが届きます。テレビや画面などの映像入力を指定します。

ID ソース (source) 部屋 (room) ラベル (label) 編集 削除
▶

音声ソース (audio_media)

映像ソースと音声ソースはメディア受信として扱い、侵入不要でコンテンツが届きます。番組音や音楽などの音声入力を指定します。

ID ソース (source) 部屋 (room) ラベル (label) 編集 削除
▶

音声認識 (STT)

音声を聴くときにテキスト変換（Speech-to-Text）を行う場合、使用するHA STTプロバイダーを指定します。 ha_stt や wyoming などのプロバイダーIDを入力してください。 空欄の場合は音量検出のみ行います（文字起こしなし）。

発話先スピーカー (speakers)

部屋ごとの発話先を設定します。HAの media_player エンティティ（TTS）か、TCPで直接PCMを送るデバイス（VoiceS3R等）を指定します。

TCP スピーカーが音声合成に使用するプロバイダー。

部屋 (room) エンティティ / ホスト ラベル (label) 編集 削除
▶

歌唱キャラクター (sing_speaker)

歌唱させる際のデフォルトのキャラクターを選択します。

VOICEVOX Songで使用可能な歌唱キャラクターの一覧です。

▶

いい感じの状態（ホームポリシー）

自律ループのとき、家の状態をここと照らし合わせて、明らかなズレだけ直します（自律操作ONのとき。OFFなら提案のみ）。自然な言葉で「こうなっていてほしい」を書いてください。例:「誰もいない部屋の照明は消えていてほしい」「暑い夜は在宅ならエアコンをつけていい」「寝ている人のいる部屋（例: 家族の寝室）は触らない」

▶

操作できる家電 (entities)

「リビングの電気つけて」のような操作を頼まれたとき、呼び方から entity_id を引くための対応表です。エンティティを選ぶと名前が自動で入ります（編集可）。同名が複数あるものは備考に「要確認」が付くので、不要な行は削除してください。

エンティティ (entity_id) 呼び方 (name) 備考 (note) 編集 削除
▶

特に気にするセンサー (sensors)

自律ループ(loop)や会話(chat)で毎回コンテキストに渡す「おもなデバイス」センサーの定義です。

▶

外部デバイス (projection_targets)

電脳体として侵入できる HA 管理外のデバイスを登録します。 ID は external:// で始まる識別子で、表示名と設置部屋を設定します。 スマートフォンなど持ち歩くデバイスは部屋を「指定なし」にしてください。

ID (external://...) 表示名 部屋 編集 削除

自律ループの間隔 (loop_schedule)

定期実行されるループの間隔と、時間帯ごとの発火確率を設定します。ここで変更した値は「preferences.json の直接編集」タブでの保存時にも常に適用されます。

定期的にバックグラウンド実行されるループの実行周期を設定します（1分以上）。

各時間帯の実行タイミングで、実際に処理を実行する確率を設定します（0〜100%）。

上の確率がこの値を下回っても、最低これだけは発火します。0なら確率どおり（例: 深夜0%にすると本当に沈黙します）。体調が悪い時間帯に完全停止して戻ってこなくなるのを防ぎたい場合だけ、小さな値（例: 2〜5）を入れてください。既定は0。

preferences.json の直接編集

アドオンの設定JSONを直接編集できます。保存前に文法チェックが行われます。

Antigravityによる音声認識

実験中機能

エージェントが「音を聴きたい」と決意したとき、次のセッションで Antigravity（Google の AI エージェント）が一時的にハーネスを担い、録音した音声ファイルの内容や雰囲気を要約・記録します。Claude Code が音声に対応したら即座に取り外せる設計です。

利用には Google アカウントへの認証が必要です。音声データは外部に送信されます。
